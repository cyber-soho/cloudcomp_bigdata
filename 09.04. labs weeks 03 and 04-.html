1Install the CLI (20 min)
🔁
No local install? Use free, browser-based shells: Google Cloud Shell, AWS CloudShell, or Azure Cloud Shell. Zero installation required.
bash — AWS CLI
# On Linux / macOS
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip

bash — Azure CLI
curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash
az --version
bash — gcloud CLI (GCP)

exec -l $SHELL
gcloud init
2Authenticate and List Regions (10 min)
bash
# AWS
aws configure
aws ec2 describe-regions --output table

# Azure
az login
az account list-locations --output table

# GCP
gcloud auth login
gcloud compute regions list

Document your observations: how many regions does your provider operate in? Which is geographically closest to your location?

3Deploy Your First Cloud Resource (30 min)
bash — AWS S3 (Simple Storage Service) bucket
aws s3 mb s3://your-unique-bucket-name-2026
echo "Hello Cloud" > hello.txt
aws s3 cp hello.txt s3://your-unique-bucket-name-2026/
aws s3 ls s3://your-unique-bucket-name-2026/
# IMPORTANT: Delete when done to avoid charges
aws s3 rb s3://your-unique-bucket-name-2026 --force
bash — Azure Blob Storage
az group create --name MyLabRG --location westeurope
az storage account create \
  --name mylabstorage2026 \
  --resource-group MyLabRG \
  --location westeurope \
  --sku Standard_LRS
# IMPORTANT: Delete when done
az group delete --name MyLabRG --yes --no-wait
bash — GCP Cloud Storage (GCS)
gsutil mb -l europe-west1 gs://your-unique-bucket-name-2026
echo "Hello Cloud" > hello.txt
gsutil cp hello.txt gs://your-unique-bucket-name-2026/
# IMPORTANT: Delete when done
gsutil rm -r gs://your-unique-bucket-name-2026
⚠️
Always clean up resources after each session. Cloud providers charge by the second for running resources. One forgotten VM or bucket can consume your entire free-tier credit overnight.

1Launch a Virtual Machine (40 min)
🔁
Cannot use a cloud VM? Use VirtualBox (free, open-source) locally, or Oracle Cloud Free Tier (always-free VMs). Be creative — research your options!
bash — AWS EC2 (Elastic Compute Cloud) t2.micro
# Create SSH (Secure Shell) key pair
aws ec2 create-key-pair \
  --key-name MyLabKey \
  --query 'KeyMaterial' \
  --output text > MyLabKey.pem
chmod 400 MyLabKey.pem

# Launch Amazon Linux 2 instance (free tier eligible)
aws ec2 run-instances \
  --image-id ami-0c02fb55956c7d316 \
  --instance-type t2.micro \
  --key-name MyLabKey \
  --count 1

# List your instances
aws ec2 describe-instances \
  --query 'Reservations[*].Instances[*].[InstanceId,State.Name,PublicIpAddress]' \
  --output table
2Connect and Explore (20 min)
bash
# Connect via SSH
ssh -i MyLabKey.pem ec2-user@<your-public-ip>

# System diagnostics inside the VM
uname -a            # Kernel information
df -h               # Disk space usage
free -m             # RAM (Random Access Memory) usage in MB (Megabytes)
lscpu               # CPU (Central Processing Unit) information
cat /etc/os-release # OS (Operating System) version
3Attach Block Storage (20 min)
bash — AWS EBS (Elastic Block Store)
# Create a 1 GB gp2 (General Purpose SSD) volume
aws ec2 create-volume \
  --availability-zone us-east-1a \
  --size 1 \
  --volume-type gp2

# Attach (replace with your IDs)
aws ec2 attach-volume \
  --instance-id i-XXXXXXXXXXXXXXXXX \
  --volume-id vol-XXXXXXXXXXXXXXXXX \
  --device /dev/sdf

# On the VM: format with ext4 and mount
sudo mkfs -t ext4 /dev/sdf
sudo mkdir /mnt/mydata
sudo mount /dev/sdf /mnt/mydata
df -h
⚠️
Terminate your VM and delete all volumes when done. "Stopping" a VM still incurs EBS storage charges. Always "terminate" (not just "stop") to avoid unexpected costs.

Hadoop — Installation and Initial Setup
Install MapReduce
⌄
🎯 Learning Objectives
  • Install Hadoop in pseudo-distributed mode on a local or cloud VM
  • Configure the four core XML (Extensible Markup Language) files
  • Run a canonical WordCount MapReduce job and inspect the output
🔁
Alternatives to local install: Docker image (docker pull apache/hadoop), Cloudera QuickStart VM, AWS EMR (Elastic MapReduce), Azure HDInsight, or GCP Dataproc. Be creative — these steps assume Ubuntu 22.04 LTS (Long-Term Support).
1Prerequisites (20 min)
bash
# Update packages
sudo apt update && sudo apt upgrade -y

# Install Java 11 (JDK — Java Development Kit)
sudo apt install -y openjdk-11-jdk
java -version

# Create dedicated hadoop OS user (best practice — isolate permissions)
sudo adduser hadoop
sudo usermod -aG sudo hadoop
su - hadoop

# Generate SSH (Secure Shell) key for passwordless local connection
ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa
cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys
chmod 0600 ~/.ssh/authorized_keys
ssh localhost   # Should connect without password prompt
2Download and Install Hadoop (20 min)
bash
# Check https://hadoop.apache.org/releases.html for the latest version
wget https://downloads.apache.org/hadoop/common/hadoop-3.3.6/hadoop-3.3.6.tar.gz
tar -xzf hadoop-3.3.6.tar.gz
sudo mv hadoop-3.3.6 /usr/local/hadoop
sudo chown -R hadoop:hadoop /usr/local/hadoop

# Set environment variables in .bashrc
echo 'export HADOOP_HOME=/usr/local/hadoop' >> ~/.bashrc
echo 'export PATH=$PATH:$HADOOP_HOME/sbin:$HADOOP_HOME/bin' >> ~/.bashrc
source ~/.bashrc

hadoop version   # Verify installation
3Configure the Four Core XML Files (30 min)
xml — core-site.xml (default filesystem URI)
<configuration>
  <property>
    <name>fs.defaultFS</name>
    <value>hdfs://localhost:9000</value>
  </property>
</configuration>
xml — hdfs-site.xml (replication factor = 1)
<configuration>
  <property>
    <name>dfs.replication</name>
    <value>1</value>
  </property>
  <property>
    <name>dfs.namenode.name.dir</name>
    <value>/home/hadoop/hadoopdata/namenode</value>
  </property>
  <property>
    <name>dfs.datanode.data.dir</name>
    <value>/home/hadoop/hadoopdata/datanode</value>
  </property>
</configuration>
xml — mapred-site.xml (use YARN as framework)
<configuration>
  <property>
    <name>mapreduce.framework.name</name>
    <value>yarn</value>
  </property>
</configuration>
xml — yarn-site.xml (NodeManager auxiliary services)
<configuration>
  <property>
    <name>yarn.nodemanager.aux-services</name>
    <value>mapreduce_shuffle</value>
  </property>
</configuration>
4Format, Start, and Verify (30 min)
bash
# Format NameNode — do this ONCE only, destroys existing data!
hdfs namenode -format

# Start HDFS (Hadoop Distributed File System) daemons
start-dfs.sh
start-yarn.sh

# Verify all JVM (Java Virtual Machine) processes are running
jps
# Expected output: NameNode, DataNode, SecondaryNameNode,
#                  ResourceManager, NodeManager

# Access Web UI (User Interface) in browser:
# http://localhost:9870  →  NameNode UI
# http://localhost:8088  →  YARN ResourceManager UI

# Upload a sample file to HDFS
hdfs dfs -mkdir -p /user/hadoop/input
echo "The quick brown fox jumps over the lazy dog" > sample.txt
hdfs dfs -put sample.txt /user/hadoop/input/

# Run the canonical WordCount MapReduce job
hadoop jar $HADOOP_HOME/share/hadoop/mapreduce/hadoop-mapreduce-examples-3.3.6.jar \
  wordcount /user/hadoop/input /user/hadoop/output

# View results
hdfs dfs -cat /user/hadoop/output/part-r-00000

2Launch PySpark and Explore the DataFrame API (30 min)
python — PySpark interactive session
from pyspark.sql import SparkSession

spark = SparkSession.builder \
    .appName("MyFirstSparkApp") \
    .master("local[*]") \
    .getOrCreate()

print(f"Spark version: {spark.version}")

# Create a DataFrame from a Python list
data = [
    ("Alice", 29, "Engineer"),
    ("Bob",   35, "Analyst"),
    ("Clara", 27, "Data Scientist"),
    ("David", 42, "Manager")
]
columns = ["Name", "Age", "Role"]
df = spark.createDataFrame(data, columns)

df.show()          # Display rows
df.printSchema()   # Print column types
df.describe().show()  # Basic statistics

# Transformation (lazy — not executed until an Action is called)
filtered = df.filter(df["Age"] < 35)
filtered.show()    # Action triggers execution

# SQL (Structured Query Language) query on a temp view
df.createOrReplaceTempView("employees")
spark.sql("SELECT Role, AVG(Age) as avg_age FROM employees GROUP BY Role").show()

spark.stop()
3Load and Analyse a Real Dataset (30 min)
💡
Download any public CSV (Comma-Separated Values) dataset from: UCI Machine Learning Repository, Kaggle Public Datasets, European Data Portal, or World Bank Open Data. Be thorough in your research — there are thousands of fascinating datasets available freely.
python — EDA (Exploratory Data Analysis)
from pyspark.sql.functions import col, count, when, isnan

# Load dataset
df = spark.read.csv("your_dataset.csv", header=True, inferSchema=True)

# Basic shape and schema
print(f"Rows: {df.count()}, Columns: {len(df.columns)}")
df.printSchema()
df.show(5)

# Count null (missing) values per column
null_counts = df.select(
    [count(when(isnan(c) | col(c).isNull(), c)).alias(c) for c in df.columns]
)
null_counts.show()

# Group and aggregate (adapt column names to your dataset)
df.groupBy(df.columns[0]).count() \
  .orderBy("count", ascending=False) \
  .show(10)
