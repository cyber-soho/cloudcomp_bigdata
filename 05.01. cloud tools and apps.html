The Mental Model of the Cloud

This section introduces the foundational concept separating cloud architecture into two distinct halves: the machinery that runs it and the software that delivers value. Understanding this distinction is crucial for navigating the broader ecosystem of cloud services.

🛠

Cloud Tools

The Vehicles & Kitchen Instruments

These are the utilities, platform features, or managed services that help engineers provision, configure, deploy, monitor, or secure cloud resources. Think of them as the ovens, knives, thermometers, and inventory systems in a restaurant's kitchen.

🍽

Cloud Applications

The Destinations & Served Dishes

These are the software products running on cloud infrastructure, accessed via the internet, delivering direct value to end-users. Think of them as the final, prepared dishes served to the restaurant's customers.

Key Abbreviations to Know:

IaC: Infrastructure as Code IAM: Identity & Access Management ETL: Extract, Transform, Load DAG: Directed Acyclic Graph SaaS: Software as a Service

Cloud Tools (Lesson 1)

Explore the six core categories of cloud tools. These are the instruments used to build, manage, monitor, and secure cloud environments. Data scientists rely on at least three of these categories in every major project.

Infrastructure Provisioning & IaC

Create and manage VMs, networks, and storage programmatically.

Infrastructure as Code (IaC)

Describing infrastructure in text files that are version-controlled like application code.

  • Repeatability: Identical environments every time.
  • Auditability: Changes tracked in version control.
  • Speed: Provision 10 VMs in one command vs. manual clicks.

Security & Identity

Shared Responsibility Model

Provider secures the infrastructure (data centres). YOU secure your data, identities, and app logic.

Principle of Least Privilege

Grant the minimum permissions necessary. Nothing more.

Bucket Access Example
Professor: Read + Write + Delete
Students: Read-Only
Guest: Denied by Default

Core Concepts

  • IAM Controls who can do what.
  • MFA Requires 2nd proof of identity.
  • VPC Isolated cloud network segment.
  • KMS Manages encryption keys.
