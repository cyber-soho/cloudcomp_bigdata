Core Conceptual Foundations

This section synthesizes Chapters 1 through 6. Interact with the elements below to review the NIST characteristics, analyze the Shared Responsibility Model visually, and explore essential architectures.

The 5 Essential Characteristics (NIST SP 800-145)

1. On-demand self-service

Spin up servers at 2 AM without calling anyone. No ticket, no waiting.

2. Broad network access

Resources are reachable from any device over standard protocols.

3. Resource pooling

One physical infrastructure serves many tenants simultaneously (multi-tenancy).

4. Rapid elasticity

Scale out for peak traffic, scale back in when demand drops.

5. Measured service

See exactly what you consumed and cost (Pay-as-you-go / PAYG).

The Shared Responsibility Model

This is the most common source of cloud breaches. The chart below illustrates how security responsibilities shift from the Customer (You) to the Provider across different service models. Hover over the segments to see examples of what is managed.

The Flat Analogy: The building management (Provider) secures the main entrance and fire escapes. You, the tenant (Customer), are responsible for locking your own front door. If your window is left open, that's a customer failure, not an infrastructure failure.

📂 Data Storage Architectures

  • Block Storage (AWS EBS) Fixed-size blocks. Low latency, high IOPS. Best for relational databases (PostgreSQL) and VM boot volumes.
  • File Storage (AWS EFS) Hierarchical folders via NFS/SMB. Best for multi-user collaboration and shared scripts.
  • Object Storage (AWS S3) Flat namespace (buckets). Unstructured data, scalable, global ID. Best for Data Lakes and ML training datasets.

⚙️ Service-Oriented Architecture (SOA)

A paradigm for organizing distributed capabilities. Services do one specific job, expose a well-defined interface, and are loosely coupled.

SOAP vs. REST
REST (Modern)
  • Architectural Style
  • JSON, lightweight
  • Stateless, easily scalable
  • Standard for Web/ML APIs
SOAP (Legacy/Enterprise)
  • Strict Protocol
  • XML, verbose
  • WS-Security built-in
  • Banking, Healthcare
