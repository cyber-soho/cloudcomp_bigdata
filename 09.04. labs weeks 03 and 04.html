Week 03: Cloud Providers & Infrastructure

In this section, you will compare AWS, Azure, and GCP, establish your first cloud deployment, and distinguish between different data storage architectures in modern cloud computing.

AWS Profile

Compute: EC2 (Elastic Compute Cloud)

Storage: S3 (Object), EBS (Block), EFS (File)

CLI Access: AWS CloudShell or local aws cli

aws s3 mb s3://my-unique-bucket-2026
echo "Hello" > file.txt
aws s3 cp file.txt s3://my-unique-bucket-2026/
Session Task Goals
  • Create an S3 bucket and upload a file.
  • Generate SSH keys and launch a t2.micro instance.
  • Create and attach an EBS gp2 volume.
  • Ensure complete cleanup to avoid billing.

Azure Profile

Compute: Azure Virtual Machines

Storage: Blob Storage (Object), Managed Disks (Block)

CLI Access: Azure Cloud Shell or local az cli

az group create --name MyLabRG --location westeurope
az storage account create --name mystorage
az group delete --name MyLabRG --yes
Session Task Goals
  • Establish a Resource Group and Storage Account.
  • Create an Ubuntu LTS Standard_B1s VM.
  • Connect via SSH and verify system resources.
  • Ensure complete cleanup to avoid billing.

GCP Profile

Compute: Compute Engine

Storage: Cloud Storage (Object), Persistent Disk (Block)

CLI Access: Google Cloud Shell or local gcloud

gsutil mb -l europe-west1 gs://my-bucket-2026
echo "Hello" > file.txt
gsutil cp file.txt gs://my-bucket-2026/
Session Task Goals
  • Deploy a Cloud Storage bucket using gsutil.
  • Provision a cloud VM instance.
  • Explore the regional architecture of the provider.
  • Ensure complete cleanup to avoid billing.

Cloud Storage Paradigms

Object Storage

Flat namespace accessed via HTTP APIs. Best for unstructured data, backups, and media.

e.g., AWS S3

Block Storage

Raw storage volumes attached directly to VMs, acting like physical hard drives for OS and databases.

e.g., AWS EBS

File Storage

Hierarchical shared file systems using standard protocols (NFS/SMB) for multiple simultaneous connections.

e.g., AWS EFS
