Basics of Cloud Computing: AWS

"The cloud is not just someone else's computer — it is the backbone of modern data science."

What is AWS?

This section introduces the core concept of Amazon Web Services (AWS), detailing its market dominance and fundamental premise. It sets the stage for why cloud computing is essential for modern data science by highlighting its scale, speed, and billing model.

  • ✔ A cloud computing platform offering 200+ services on-demand over the internet.
  • ✔ Billed by actual usage — no upfront hardware purchase.
  • ✔ Spin up a virtual server in under 2 minutes.
  • ✔ Born in 2006 from Amazon solving its own holiday traffic scaling problems.

Global Market Position

AWS holds the largest share of the global cloud Infrastructure as a Service (IaaS) market, ahead of Microsoft Azure and Google Cloud Platform.

Source: Gartner & Synergy Research Group

Key Takeaway

Most data science workloads are "bursty." The break-even point is ~7,600 hours (~317 days of 24/7 use). For typical graduate student usage (~12 hours/week), cloud wins decisively.

Core Service Architecture

This interactive explorer categorizes the vast AWS ecosystem into manageable domains. Select a category below to reveal the foundational services, their analogies, and their specific roles in a data scientist's toolkit.

Compute Services

The foundational engines providing raw processing power.

EC2 (Elastic Compute Cloud) Core

Virtual servers by the hour/second. Pick CPU, RAM, GPU, OS for full control.

Analogy: Renting a furnished apartment

Lambda

Serverless computing. Run functions without managing servers. Pay per millisecond of execution — zero idle cost.

Analogy: Hiring a freelancer per task

ECS / EKS

Run Docker (ECS) or Kubernetes (EKS) containers for portable, reproducible deployments.

Analogy: Standardized shipping containers

Storage Services

Where your data lives, from active files to deep archives.

S3 (Simple Storage Service) The Backbone

Object storage for any file. 99.999999999% durability (eleven 9s). Nearly every AWS workflow starts and ends here.

Analogy: Infinite Google Drive

EBS (Elastic Block Store)

SSD drives directly attached to your EC2 instances.

Analogy: USB external hard drives

Glacier

Ultra-cheap cold storage for long-term archival.

Analogy: Off-site storage unit

Database Services

Structured and unstructured data management at scale.

RDS (Relational Database Service)

SQL (MySQL, PostgreSQL). Best for structured data and transactions. AWS handles backups, patches, and scaling.

DynamoDB

NoSQL key-value database. Provides single-digit millisecond latency at any scale.

Redshift

Data warehouse with columnar storage optimized for complex SQL analytics across billions of rows.

Machine Learning

The most relevant tools for data scientists to build and deploy models.

SageMaker

Full ML lifecycle platform: build, train, deploy. Includes Studio (cloud IDE with Jupyter), distributed training, and 1-click HTTPS endpoints.

Comprehend

NLP service to find sentiment, entities, and topics in text.

Rekognition

Computer vision service for identifying objects, faces, and text in images and video.

Networking & Security

The underlying infrastructure that keeps everything connected and secure.

Networking

  • VPC: Your isolated private network in the cloud.
  • Route 53: DNS service (Domain to IP translation).
  • CloudFront: CDN caching content at 400+ edge locations.

Security

  • IAM: Identity & Access Management (Who can access what).
  • KMS: Encryption key management.
  • CloudTrail: Audit log of every API call made in your account.

Pricing Models & Cost Estimation

Smart pricing choices can save 70%+. Compare On-Demand pricing versus using Spot Instances and the Free Tier for a typical monthly thesis project (50GB S3, 80hrs GPU, 24/7 API).

On-Demand: No commitment, predictable, but highest cost per hour. Best for unpredictable workloads.
Spot Instances: Up to 90% discount. AWS can reclaim with 2-min warning. Perfect for ML training if you use checkpointing!

Security & Data Workflow

Understand the Shared Responsibility Model and trace a real-world data pipeline. This area clarifies where your security obligations lie and visualizes how data flows from ingestion to machine learning deployment.

The Shared Responsibility Model

Misconfigured IAM is the #1 cause of cloud security incidents. Know your boundaries.

🏢

AWS Secures "OF" the Cloud

  • ✔ Physical data centers & hardware
  • ✔ Global networking infrastructure
  • ✔ Hypervisor layer
  • ✔ Managed service infrastructure

Analogy: AWS builds the apartment building, installs locks on the main doors, and provides fire exits.

🔑

YOU Secure "IN" the Cloud

  • ✔ Customer data & encryption choices
  • ✔ IAM Configuration (Users, Roles, Policies)
  • ✔ Network settings (VPC, firewalls)
  • ✔ Application code & dependencies

Analogy: You secure your specific apartment—who has keys, what's inside, and remembering to lock the door.

THE GOLDEN RULE: Principle of Least Privilege — Every entity gets ONLY the minimum permissions needed.
