The Scale Problem

Why do traditional systems crack under Big Data? This section visually demonstrates the difference between Vertical Scaling (buying bigger hardware) and Horizontal Scaling (distributing workload across commodity machines). Horizontal scaling is the foundation of Hadoop and Spark.

Vertical Scaling (Scale Up)

Limits: CPU, RAM, Cost

VS

Horizontal Scaling (Scale Out)

1 node active. Fault tolerance: None.

Why Vertical Fails

  • ▶ Hardware limits (finite RAM/CPU)
  • ▶ Cost explosion (doubling capacity triples price)
  • ▶ Single Point of Failure (SPOF)
  • ▶ I/O network bottlenecks

Why Horizontal Wins

  • ▶ Infinite scalability via network
  • ▶ Uses cheap commodity hardware
  • ▶ Inherent fault tolerance (replication)
  • ▶ Parallel processing (MapReduce)

Hadoop vs Apache Spark

Explore the evolution of Big Data processing. Hadoop revolutionized storage (HDFS) and batch processing (MapReduce) on disk. Spark emerged to solve MapReduce's latency by keeping Resilient Distributed Datasets (RDDs) in memory, making it up to 100x faster for iterative machine learning workloads. They are often used together in layered architectures.

Conceptual Performance Benchmarks

When to use Hadoop: Petabytes of archival data, batch ETL, cold-storage, budget prioritizes disk storage over speed.
When to use Spark: Fast iterative ML training, real-time streaming, unified batch/SQL/ML engine, Python/Scala teams.

Disk Hadoop Architecture

MapReduce (Processing) Two phases: Map (transform locally) -> Shuffle & Sort -> Reduce (aggregate). Writes to disk between steps.
YARN (Resource Management) Yet Another Resource Negotiator. Manages cluster CPU/RAM and schedules jobs.
HDFS (Storage) Hadoop Distributed File System. Splits files into 128MB blocks, replicates 3x across DataNodes.

RAM Spark Architecture

Driver & Actions (Execution) Lazy evaluation. Transformations build a DAG (Directed Acyclic Graph). Actions trigger execution.
Cluster Manager & Executors Connects via SparkContext. Worker nodes execute tasks and store RDDs in memory.
Unified Libraries Spark SQL (DataFrames), Spark Streaming, MLlib (Machine Learning), GraphX.
