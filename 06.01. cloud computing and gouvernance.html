Cloud Computing Essentials

This section introduces the foundational concepts of cloud computing, defining what it is, its core characteristics, and the financial shift it brings to organizations. Understanding these basics is critical before architecting or governing cloud environments.

📖 NIST Definition

"On-demand network access to a shared pool of configurable computing resources that can be rapidly provisioned and released with minimal effort."

Analogy: Instead of building a power plant, you plug into the grid and pay per kilowatt-hour.

⭐ Five Essential Characteristics

  • 1
    On-demand self-service: Provision resources without human intervention.
  • 2
    Broad network access: Available from any device via standard protocols.
  • 3
    Resource pooling: Multi-tenant; many customers share hardware.
  • 4
    Rapid elasticity: Scale up/down in minutes, not months.
  • 5
    Measured service: Pay only for what you consume.

💰 Cloud Economics: CapEx vs. OpEx

Cloud shifts IT spending from capital investments (buying servers) to operational expenses (renting resources). Use the chart below to compare the financial impact.

Rule of Thumb: Cloud wins for variable workloads (< 70% utilisation). On-prem wins for sustained, maxed-out hardware (> 70% utilisation).

Service & Responsibility

Selecting a cloud service model involves a direct trade-off between control and convenience. This section breaks down IaaS, PaaS, and SaaS, and critically visualizes how security responsibilities shift depending on your choice.

Infrastructure as a Service

You rent raw compute, storage, and networking.

🍕 Analogy: Renting a bare kitchen

  • You manage: OS, runtime, applications, data.
  • Provider manages: Hardware, networking, virtualisation.
  • Best for: Custom environments, GPU-heavy ML workloads.

Key Providers

AWS EC2 Azure VMs Google Compute Engine

Platform as a Service

You deploy code; the platform handles the rest.

🍕 Analogy: A fully equipped kitchen

  • You manage: Application code, data.
  • Provider manages: OS, runtime, middleware, scaling.
  • Best for: Rapid prototyping, managed ML training, web apps.

Key Providers

Google App Engine Heroku Azure App Service

Software as a Service

Ready-to-use applications over the internet.

🍕 Analogy: Ordering delivery

  • You manage: Your content and configuration.
  • Provider manages: Everything else.
  • Best for: End-user tools, no-code analytics, collaboration.

Key Providers

Google Workspace Salesforce Dropbox

🛡️ The Shared Responsibility Model

"The cloud is secure — but YOUR configuration might not be."

Real-World Consequence: Misconfigured S3

The #1 cloud security incident occurs when a team uploads sensitive data to AWS S3 and sets the bucket to "public". AWS infrastructure is never breached — the configuration is the customer's job. You ALWAYS own responsibility for your data.

Cloud Governance

Governance is the set of policies, processes, and controls that manage a cloud environment to be secure, compliant, and cost-effective. Think of cloud computing as a sports car, and governance as the traffic laws, insurance, and driving instructor.

💸 Cost Governance

The $12,400 Weekend:
An intern launched 8× NVIDIA A100 GPUs for a sweep, forgot to shut down, resulting in $12k charges for idle time.
  • ✅ IAM Policy: Limit to 2 GPUs
  • ✅ Budget Alert: SMS alert at $500
  • ✅ Auto-shutdown: Stop after 2h idle

🔑 IAM: Identity & Access Management

The Golden Rule: Least Privilege. Give every user the minimum permissions they need.

Role Permitted Denied
ML Engineer Training instances, data read IAM changes, billing
Data Analyst Data read, queries GPU instances, deletes
Intern Read dashboards only Write to anything
