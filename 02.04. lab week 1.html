Virtualisation and Containerisation

Understand how the cloud abstracts hardware — and how containers changed deployment forever.

⏱ ~90 min
2.1
Understanding Virtualisation & Hypervisors
~25 min
▼

Virtualisation is the process of creating a software-based (logical) representation of physical hardware. A hypervisor (also called a VMM) is the software layer enabling this.

TypeHow It WorksExamplesCloud Relevance
Type 1
(Bare-metal)		 Runs directly on hardware — no host OS needed VMware ESXi, Microsoft Hyper-V, Xen Project Used by all major cloud providers — maximum performance
Type 2
(Hosted)		 Runs on top of a conventional OS Oracle VirtualBox, VMware Workstation Developer workstations, local testing
💡 Your task: Draw a side-by-side hypervisor comparison diagram. Explain (3–5 sentences) why cloud providers use Type 1. Reference: Xen Project (open-source Type 1)
2.2
Containers vs Virtual Machines — The Comparison
~25 min
▼

Containers package an application with its libraries and configuration, but share the host OS kernel — making them lighter, faster, and more portable than full VMs.

CriterionVirtual MachinesContainers
Startup timeMinutesSeconds or milliseconds
Typical image sizeGigabytesMegabytes
OS isolationFull (separate guest OS per VM)Partial (shared host kernel)
PortabilityModerate — depends on hypervisorHigh — runs anywhere with container runtime
Typical use caseFull OS environments, legacy appsMicroservices, CI/CD pipelines, cloud-native apps

4.4
Hands-On — Build a Minimal REST API
~20 min
▼

Expose a tiny REST API. Any language or framework you prefer — the example below uses Python + Flask (open-source). Research and use alternatives: Node.js + Express | Go stdlib | Rust + Actix

PYTHON — app.py 
# Minimal REST API (Application Programming Interface) — educational demo
from flask import Flask, jsonify

app = Flask(__name__)

# In-memory store — in production, this would be a cloud data service
cloud_services = [
    {"id": 1, "name": "Compute", "type": "IaaS"},
    {"id": 2, "name": "Object Storage", "type": "IaaS"},
    {"id": 3, "name": "Managed Kubernetes", "type": "PaaS"},
]

@app.route("/services", methods=["GET"])
def get_services():
    """Return all cloud service entries."""
    return jsonify(cloud_services)

@app.route("/services/<int:service_id>", methods=["GET"])
def get_service(service_id):
    service = next((s for s in cloud_services if s["id"] == service_id), None)
    if service:
        return jsonify(service)
    return jsonify({"error": "Service not found"}), 404

if __name__ == "__main__":
    # DEBUG mode — NEVER use debug=True in a real production deployment
    app.run(debug=True, port=5000)
BASH — Test the API 
pip install flask
python app.py

# In a second terminal — using curl (command-line HTTP tool)
curl http://localhost:5000/services
curl http://localhost:5000/services/2
